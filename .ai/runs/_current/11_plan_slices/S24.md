# Slice S24: Giỏ hàng Session & Giao diện Giỏ hàng

## 1. Mục tiêu
Cho phép người dùng tương tác với giỏ hàng: thêm, bớt sản phẩm, xoá sản phẩm và xem tổng số lượng tại thanh Navbar. Hệ thống sử dụng Session để lưu thông tin tạm.

## 2. Các bước thực hiện

### Bước 1: API Router & Controller (Backend)
- **File:** `src/modules/public/cart/cart.route.js` & `cart.controller.js` (Tạo mới).
- **Endpoints:**
  - `POST /cart/add`: Body cần `{ product_id, quantity }`. Lấy giỏ từ `req.session.cart`. Nếu sp đã tồn tại -> cộng dồn lên. Nếu chưa -> push vào mảng.
  - `POST /cart/update`: Body `{ product_id, quantity }`. Update lại quantity. Nếu quantity = 0, báo lỗi hoặc gọi qua Remove.
  - `POST /cart/remove`: Xóa item khỏi session.
  - `GET /cart/count`: Trả về tổng quantity để update Badge trên Navbar.
  - `GET /cart`: Render view `cart.ejs`.
- **Logic lấy giá (`cart.service.js`):** Mapping `product_id` từ session để lấy DB (tránh việc lưu cục giá tiền ở session). Join `Product` để lấy `name, slug, p.images[0], price, sale_price, stock_quantity`. Tính thành tiền mỗi dòng.

### Bước 2: Frontend JS Handling (`public/js/cart.js`)
- **Action:** Dùng Fetch API để gọi các tuyến đường POST trên. 
- Xử lý SweetAlert2 hoặc Toastr (nếu có) để báo thêm vào giỏ thành công. Tự động fetch `GET /cart/count` hiển thị lên thẻ `<span class="badge" id="cart-badge">`.
- Gắn file này vào `main.ejs` nếu muốn dùng Global.

### Bước 3: UI Nút Thêm Vào Giỏ
- **Shop & Detail:** Sửa code nút Add to cart trong `shop.ejs`, `detail.ejs`, hoặc các trang Home (Featured) để bắn hàm `addToCart(id, qty)` thay vì Alert báo chờ.

### Bước 4: Trang Giỏ Hàng (`src/views/public/cart/index.ejs`)
- Hiển thị danh sách sản phẩm dạng bảng (Image, Tên, Giá, Input Số lượng +- , Thành tiền, Nút Xóa rác).
- Sidebar: Tổng tiền tạm tính = Tổng (Price * Qty). Kèm nút "Proceed to Checkout".

## 3. Tech Stack
- View: EJS
- BackEnd: Express Session, Prisma Client
- HTTP Client: JS Native `fetch`

# Slice S23 — Admin Multi-Image Upload Bug Fix
- Goal: Sửa lỗi ghi đè ảnh, hỗ trợ tải lên tối đa 5 ảnh trên Admin page.

## Checklist chi tiết
- [ ] Admin Interface (`views/admin/pages/products/create.ejs` & `edit.ejs`):
    - Cập nhật ô input file hỗ trợ multiple (`<input type="file" name="images" multiple max="5" />`), UI drag-drop hoặc hiển thị nhiều preview.
    - Trong Edit page: Hiển thị danh sách ảnh đang có kèm nút "Xóa" hoặc "Đặt Thumbnail" (Xử lý ajax/form hidden input `removeImages[]`).
- [ ] Upload Middleware (`utils/upload.js` hoặc tương đương):
    - Multer route: Thay vì `.single('image')`, dùng `.array('images', 5)` cho route upload.
- [ ] Product Controller (`controllers/admin/ProductController.js`):
    - Cập nhật DB: Lưu mảng kết quả vào `ProductImage` với `sort_order` tương ứng.
    - Xử lý mảng Cloudinary URLs: `req.files` thay vì `req.file`.
    - Fix logic update: Không xóa ảnh cũ nếu không có yêu cầu xóa.
- [ ] DB Seeding (Optional): Test seed file nếu cần kiểm thử.

## Definition of Done
- Admin upload lên 3 ảnh cùng lúc hệ thống lưu đủ 3 ảnh vào DB.
- Khi update sản phẩm thêm 1 ảnh mới, hệ thống có chứa 4 ảnh.
- UI hiển thị đúng nhiều ảnh preview trước khi submit.

## Verification Commands

- Đăng nhập Admin, vào `/admin/products/create` submit form 3-4 ảnh và check DB.

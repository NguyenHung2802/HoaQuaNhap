# Slice S28: Order Detail & Trạng thái Vận hành (Restock Logic)

## 1. Mục tiêu
Hỗ trợ admin xem thông tin cặn kẽ 1 đơn hàng cụ thể, các biến động kho tương ứng và điều chỉnh được trạng thái đơn. Đặc biệt, logic bảo toàn tồn kho (Restock) sinh tử phải ổn định.

## 2. Các bước thực hiện

### Bước 1: Routing Backend & Tải giao diện
- **Route:** `GET /admin/orders/:id`.
- Fetch `order` bao gồm `items (sản phẩm, giá)` và `status_logs` (thời gian đổi trạng thái). Truyền xuống template view.

### Bước 2: View Chi tiết đơn (`views/admin/orders/detail.ejs`)
- Giao diện 2 cột: 
    - Cột 7: Grid Items và Total Amount, Note KH. Timeline lịch sử `OrderStatusLog` (vertical timeline EJS).
    - Cột 5: Form Contact nhận hàng + Dropdown "Cập nhật trạng thái" kèm nút Save.

### Bước 3: Action Update Status (Cực Kì Quan Trọng)
- **Route:** `POST /admin/orders/:id/status`. Body `{ new_status, note }`
- **Validation:** Lấy Status cũ ra check. Nếu `old_status` == 'cancelled' hoặc 'completed' mà cố đổi lại pending -> Báo lỗi. (hoặc nếu yêu cầu đổi về được thì phải cẩn thận handle). Tạm thời không cho đi ngược nếu đã done/cancel.
- Dùng `prisma.$transaction`:
    1. Update Order `order_status`. Nếu = 'completed' thì gán thêm `payment_status` = 'paid'.
    2. Ghi một Dòng vào `OrderStatusLog` lưu user đăng nhập bằng Session admin.
    3. **Nếu đổi sang `cancelled`**:
       - Iterate over `OrderItem`
       - Tăng tương ứng `Product.stock_quantity = stock_quantity + item.quantity`
       - Gán `InventoryLog` `type` ='order_add'.
- Trả về Flash Thành công và reload trang chi tiết.

# Slice S1: Database & Model (Unified User)

- **Goal**: Cập nhật Prisma Schema để hỗ trợ bảng User hợp nhất với field `role`. Di cư (Migrate) database và nạp lại dữ liệu seed mẫu cho Admin.
- **Tham chiếu**:
  - `AGENTS.md`
  - `.ai/runs/_current/00_ba.md`
  - `.ai/runs/_current/10_plan.md`
  - `prisma/schema.prisma`

## Checklist Chi tiết
- [ ] Mở file `prisma/schema.prisma`.
- [ ] Cập nhật model `User` để có trường `role` (enum hoặc string) và `status`.
- [ ] Thêm các trường cần thiết cho đơn hàng khách nếu chưa có.
- [ ] Chạy lệnh `npx prisma migrate dev --name unified_user`.
- [ ] Cập nhật `prisma/seed.js`: 
  - [ ] Thêm thư viện `bcryptjs`.
  - [ ] Tạo Admin user với mật khẩu `admin123` được mã hóa.
- [ ] Chạy lệnh `npx prisma db seed`.

## Files/Modules dự kiến
- `prisma/schema.prisma`
- `prisma/seed.js`
- `node_modules/@prisma/client` (tự động generate)

## DoD (Acceptance Criteria)
- [ ] Có thể thấy bảng `User` trong Postgres.
- [ ] Tài khoản Admin được tạo ra với `role` là "admin".
- [ ] Mật khẩu trong DB không phải văn bản thô (đã hash).

## Verification Commands
- `npx prisma migrate dev --name unified_user`
- `npx prisma db seed`
- `npx prisma studio` (Để kiểm tra dữ liệu thủ công)
